# Distill: Personal Intelligence Platform — Design

## Vision

Distill is a personal intelligence layer. It ingests everything you read, build, and think — then synthesizes it, surfaces connections, builds your voice, tracks your learning trajectory, and discovers what you should read next.

The product is a **Daily View dashboard** you open every morning. Not a data dump. A personalized newspaper built from your own signal.

## The Flywheel

```
Read → Distill synthesizes → Distill discovers new content → You read that → Better synthesis → Better discovery
```

Each cycle makes the system more valuable. The more you use it, the smarter it gets about what matters to you.

## Three Core Capabilities

### 1. It Thinks For You
Surfaces connections you haven't noticed. Links something you read today to something you built last week. Notices when a topic keeps appearing across your reading and sessions. The "Connection" insight in the Daily View is the flagship expression of this.

### 2. It Builds Your Voice
Learns from your edits over time (voice memory). Produces content across LinkedIn, X, and Ghost that sounds increasingly like you. The more you edit drafts, the better the next draft gets. You become a prolific, consistent creator without the grind.

### 3. It Makes You Smarter
Shows your learning trajectory. Topics trending up in your attention. Blind spots. How your thinking has evolved. The "Learning Pulse" visualization is a mirror for your intellectual growth.

## Daily View — The Main Product Surface

The Daily View replaces the current Dashboard tab as the front door. Six sections, designed to be consumed in under 5 minutes.

### Section 1: "3 Things Worth Knowing"
- Top 3 reading highlights from yesterday
- 2-3 sentences each, ranked by interestingness
- Reading only — no coding sessions mixed in
- Source links included for deep dives
- Generated by the morning pipeline job

### Section 2: "Connection"
- One AI-generated insight linking something new to something from your past
- Examples: "This article about ambient agents connects to the pre-flight gate pattern you wrote about 2 weeks ago" or "You've read about interpretability 4 times in 10 days — here's the thread forming"
- Requires cross-referencing current intake against historical entries (unified memory, journal history, blog posts)
- This is the hardest section to build well and the most valuable

### Section 3: "Explore Next" — The Discovery Engine
- Based on your topics, interests, trajectory, and recent reading
- High-quality content recommendations: articles, YouTube videos, papers, podcasts
- Adjacent topics you haven't explored yet
- What others in your space are doing
- People to follow, tools to check out
- Sources: web search, YouTube API, RSS discovery, curated feeds, social signal
- Quality filter: no listicles, no SEO spam — substantive content only
- Refreshes daily, learns from what you click vs. skip

### Section 4: "Ready to Post"
- Draft posts for each platform, generated from reading (not sessions)
- LinkedIn: short-form insight post, applying voice rules
- X/Twitter: thread or single post format
- Ghost: blog candidate when there's enough material for a full piece
- Edit inline in the dashboard
- One-click push to Postiz (LinkedIn, X) or Ghost API
- Voice rules applied automatically, banned patterns caught before you see the draft

### Section 5: "What You Built"
- Below the fold — session highlights from yesterday
- Key commits, time spent, tools used
- Separated from reading to avoid the "sessions drown everything" problem
- Linked to journal entry for full detail

### Section 6: "Learning Pulse"
- Small visualization showing topic attention over 2-4 weeks
- Topics trending up (deepening), cooling (fading), emerging (new)
- Based on reading intake + session activity + blog topics
- Data already partially exists in the briefing system (areas, learning, threads)

## Architecture

### What Exists Today
- Intake pipeline: RSS, browser, session, manual → ContentItem → digest
- Journal pipeline: sessions → daily journal entry
- Blog pipeline: journal + intake → weekly/thematic blog → publishers
- Voice memory: extraction from edit history, rule application to prompts
- Web dashboard: Hono API + React + Tailwind + TanStack Router
- Briefing system: areas, learning, risks, open threads
- Concept graph: visualization of briefing data

### What Needs to Be Built

**Backend (Python — `src/`):**

1. **Reading-only digest** — A new synthesis pass that excludes sessions. Input: only RSS, browser, manual, discovery items. Output: ranked highlights with interestingness scores. Stored as a new artifact type alongside the full digest.

2. **Connection engine** — Cross-references today's reading against: unified memory threads, past journal entries, past blog posts, past intake digests. Uses embeddings (already have sentence-transformers) for semantic similarity. Surfaces the single strongest connection with an explanation. Stored as part of the daily brief.

3. **Discovery engine** — Based on extracted topics/entities from recent reading (last 7 days), performs:
   - Web search for high-quality articles on active topics
   - YouTube API search for relevant videos
   - RSS feed discovery (find new feeds covering your interests)
   - Social signal (who's writing about the same things)
   - Quality filtering (length, source reputation, freshness)
   - Dedup against already-read items
   - Output: 5-8 curated recommendations with summaries
   - Stored as discoverable items that feed back into the next intake cycle

4. **Auto-draft generation** — After the reading-only digest runs, automatically generate:
   - LinkedIn draft (apply voice rules, < 900 chars, reading-first)
   - X draft (shorter format, different voice adaptation)
   - Ghost candidate (when 2+ related topics form a narrative arc)
   - Store in `insights/blog/postiz/` and a new `insights/drafts/` directory

5. **Learning trajectory** — Aggregate topic tags across intake items over 2-4 weeks. Compute: trending up (increasing frequency), cooling (decreasing), emerging (new in last 3 days), stable (consistent). Store as part of briefing data.

**Frontend (TypeScript — `web/`):**

6. **Daily View page** — New route `/daily` that becomes the default landing page. Six-section layout as described above. API endpoints for each section. Mobile-responsive.

7. **Inline post editor** — Edit draft posts directly in the Daily View. Markdown preview. Character count. Platform-specific formatting hints. Push-to-Postiz and push-to-Ghost buttons.

8. **Discovery interaction** — Click to read, save for later, or dismiss. Feedback loop: clicks and saves inform future recommendations.

9. **Learning Pulse widget** — Small chart (sparklines or area chart) showing topic attention over time. Reuses topic data from briefing system.

### Pipeline Changes

The morning job currently runs: sessions → journal → intake → blog (weekly).

New pipeline:
```
sessions → journal → intake (full) → reading-only digest → connection engine → discovery engine → auto-drafts → briefing update
```

The blog pipeline stays weekly. The new steps (reading-only, connection, discovery, auto-drafts) run daily.

## Content Strategy Defaults
- Daily posts source from reading list first, not coding sessions
- No TroopX mentions unless explicitly requested
- Subtle learning angle — what's interesting from what was read
- Voice rules applied to all auto-generated drafts

## Session Capture Gap
Current issue: not all coding sessions are being captured. Needs investigation:
- Are some session directories not being scanned?
- Are sessions below a minimum length/activity threshold being filtered out?
- Are sessions from other tools (non-Claude, non-Codex) being missed?

## What This Replaces / Changes
- Dashboard tab → Daily View becomes the default landing page
- Current digest (34K words, 50 items) → still generated but not the primary consumption surface
- Manual LinkedIn workflow (Claude Code conversation) → automated draft generation
- No discovery/recommendation → active discovery engine feeding the reading loop

## Success Criteria
- User opens the Daily View every morning (retention)
- Time from "open dashboard" to "posts scheduled" is under 5 minutes
- Discovery recommendations lead to actual reading (click-through)
- Voice accuracy improves over time (fewer edits per draft)
- Session capture is complete (all sessions appear in journal)

## Phasing (Suggested)

**Phase 1: Daily View + Reading-Only Digest**
- Build the Daily View page layout
- Separate reading items from sessions in intake
- Generate "3 Things Worth Knowing" as a new artifact
- Move auto-draft generation into the pipeline (LinkedIn first)

**Phase 2: Connection Engine + Learning Pulse**
- Cross-reference today's reading against history
- Topic trajectory tracking
- Learning Pulse visualization

**Phase 3: Discovery Engine**
- Web search integration
- YouTube API integration
- Quality filtering and dedup
- Feedback loop from user interaction

**Phase 4: Full Content Studio**
- Inline editing for all platforms
- One-click publishing to Postiz and Ghost
- X/Twitter draft generation
- Ghost blog candidate generation
